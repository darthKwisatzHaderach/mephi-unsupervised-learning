## 🔷 Принципы оформления

### 🧭 Нарратив и логика
| Принцип | Как реализовать |
|---------|----------------|
| **Одна ячейка — одна идея** | Код: одна функция / один шаг пайплайна. Markdown: одно объяснение. |
| **Markdown перед кодом** | Сначала объясняете *что и зачем*, потом показываете *как*. |
| **Заголовки-якоря** | Используйте `##`, `###` для навигации. Избегайте вложенности >3 уровней. |
| **Связки между разделами** | В конце раздела — 1 предложение-переход: *«Теперь, когда данные очищены, перейдём к моделированию…»* |

### 📝 Документация кода
```python
def enforce_si_invariant(
    predictions: np.ndarray,
    alpha: float = 0.35,
    eps: float = 1e-3,
) -> np.ndarray:
    """
    Применяет мягкий физический инвариант SI ≈ CC50 / IC50.
    
    Мотивация:
    - На тренировочных данных выполняется соотношение SI = CC50 / IC50 с погрешностью ~1%.
    - Прямая замена SI на ratio ухудшает обобщение (переобучение на шум в знаменателе).
    - Бленд с весом α позволяет сохранить гибкость модели, но «подтянуть» предсказания к физике.
    
    Формула:
        SI_final = α * SI_direct + (1 - α) * CC50 / max(IC50, ε)
    
    Args:
        predictions: np.ndarray формы (n_samples, 3) — [IC50, CC50, SI_direct]
        alpha: вес прямого предсказания в финальном бленде (0 ≤ α ≤ 1)
        eps: численная стабильность для деления (избегаем деления на 0)
    
    Returns:
        np.ndarray формы (n_samples, 3) — скорректированные предсказания, все значения ≥ 0
    
    Примечание:
        Функция детерминирована, не зависит от состояния RNG.
    """
    result = predictions.copy()
    ic50_safe = np.clip(result[:, 0], eps, None)
    si_ratio = result[:, 1] / ic50_safe
    result[:, 2] = alpha * result[:, 2] + (1.0 - alpha) * si_ratio
    return np.clip(result, 0, None)
```

### 🔁 Воспроизводимость
- **Все импорты** — в первой ячейке.
- **Все гиперпараметры и пути** — вынесены в константы в начале.
- **`RANDOM_STATE`** — объявлен глобально, используется везде, где есть стохастичность.
- **Версии пакетов** — в конце ноутбука:

## 🔷 Визуализация: правила для исследований

> Графики — не украшение, а инструмент аргументации. Каждый график должен отвечать на вопрос.

### 📊 Чек-лист качественного графика
```text
[ ] Подписаны оси: название + единицы измерения (например, "IC50, мМ")
[ ] Есть заголовок или подпись под графиком: что сравниваем / какой вывод
[ ] Легенда: если несколько линий/точек — понятно, что есть что
[ ] Цвет: доступная палитра (учёт дальтонизма), не >6 категорий без паттернов
[ ] Масштаб: логарифмический, если данные сильно скошены (указывать в подписи!)
[ ] Источник данных: "train", "test", "out-of-fold", "synthetic" — явно
[ ] Размер: график читается без зума (ширина ~800px для отчёта)
```

### ❌ Эксперимент #7: Стеккинг регрессоров (Ridge + RF + XGBoost)

**Гипотеза**:  
Ансамблирование разнородных моделей через метамодель (Ridge) улучшит обобщение за счёт компенсации ошибок.

**Реализация**:  
- Базовые модели: Ridge, RandomForest, XGBoost (все обучались на одних и тех же признаках).  
- Метамодель: Ridge на out-of-fold предсказаниях (5-fold).  
- Финальное предсказание: на test через предсказание базовых моделей → метамодель.

**Результат**:  
| Подход | CV RMSE (mean ± std) | Public LB |
|--------|---------------------|-----------|
| Лучший одиночный (Clustering+HGB) | 298.1 ± 3.8 | 292.06 |
| Стеккинг (3 модели) | **301.4 ± 5.2** | 295.81 |

**Почему не сработало**:  
1. Малый размер выборки (n=751) → метамодель переобучается на шум в out-of-fold предсказаниях.  
2. Высокая корреляция между базовыми моделями (все используют одни признаки) → мало разнообразия для блендинга.  
3. Увеличение дисперсии предсказаний не компенсировалось снижением смещения.

**Вывод**:  
Для малых датасетов предпочтительнее *простой взвешенный бленд* с фиксированными весами, чем обучение метамодели. Стеккинг стоит пробовать при n > 5000 или при использовании сильно разнородных признаков (текст + графы + таблица).

**Статус**: 🚫 Закрыто, не включать в финальный пайплайн.
