## 1) Что было сделано в базовом пайплайне

- Загружены `train/test/sample_submission`, проверены размеры и структура колонок.
- Учтена несогласованность имен таргетов:
  - в train: `IC50, mM`, `CC50, mM`, `SI`
  - в submission: `IC50`, `CC50`, `SI`
- Проверены пропуски:
  - `X_train`: 24 NaN
  - `X_test`: 12 NaN
  - `y_train`: 0 NaN
- Проверены типы данных: все признаки числовые.
- Проверены распределения таргетов: выраженная правосторонняя асимметрия и длинные хвосты.
- Принято решение обучать в лог-шкале (`log1p`) с обратным преобразованием (`expm1`).

## 2) Ключевые решения и результаты

### Бейзлайн CatBoost

- Модель: target-wise CatBoost (отдельная модель на каждый таргет).
- Метрика: средний RMSE по 3 таргетам.
- Сравнение raw vs log:
  - `RAW CV: 486.68043 ± 137.59825`
  - `LOG CV: 473.95213 ± 167.36430`
- Выбран режим `log1p`.

### Усреднение по seed

- Добавлено усреднение предсказаний по нескольким seed (снижение variance).
- Небольшой положительный эффект на leaderboard.

### Target-wise tuning CatBoost

- Подобраны параметры отдельно для `IC50/CC50/SI`.
- Дало прирост относительно раннего бейзлайна.

### Feature Importance + SHAP + top-k

- Посчитаны feature importance по CatBoost.
- Посчитаны SHAP-важности для `SI`.
- Выполнен отбор top-k признаков по aggregated ranking.
- Проверка по CV:
  - `top-60: 477.31870 ± 163.23383`
  - `top-90: 474.32910 ± 165.71511`
  - `top-120: 472.42535 ± 166.34214`  <- лучший
  - `top-150: 474.03069 ± 164.19053`
  - `top-180: 472.63745 ± 166.69251`
  - `top-210: 474.23903 ± 165.61606`
- Лучший `k = 120`.
- На обновленном тюнинге CatBoost + top-k получен лучший score этого трека: **312.21848**.

### Кластерный трек (принципиально новый подход)

- Введен отдельный подход: `KMeans`-кластеры как дополнительные one-hot признаки к исходным дескрипторам.
- Ключевой результат:
  - `submission_clustering.csv` -> **306.26468** (лучший общий результат на данный момент).
- Дополнительно проверены:
  - `submission_clustering_tuned.csv` (расширенный поиск `k` + модель) — без улучшения относительно 306.26.
  - `submission_clustering_blend_topk.csv` — без улучшения относительно 306.26.

## 5) Где основные точки роста сейчас

1. **Углубить именно кластерный подход (главный источник прироста)**
   - Расширить пространство кластерных признаков: добавить не только one-hot, но и расстояния до центроидов.
   - Пробовать разные `k` и random seeds KMeans в одном ансамбле.

2. **Локальный blend только вокруг победного трека**
   - Блендить `submission_clustering.csv` с лучшим CatBoost-topk, но через узкую сетку весов около кластера (`w_cluster ~ 0.85..0.98`).
   - Не делать широкие blend-сетки с заведомо слабыми моделями.

3. **Точечная стабилизация SI в кластерном треке**
   - Проверить per-target модели в кластерах (или кластерно-зависимые веса) только для `SI`.
   - Оценивать отдельно вклад в RMSE(SI), затем смотреть общий score.

4. **Минимизировать регресс по baseline**
   - Любая новая гипотеза принимается только если beat текущего best `306.26468` на leaderboard.
